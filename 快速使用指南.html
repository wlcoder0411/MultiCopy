📋 MultiCopy 快速使用指南

多段连续复制 · 队列按序/选择性粘贴 · 置顶保护 · 内容分组 · 暗亮双主题 · 常驻置顶窗口

核心流程

1连续复制在任意应用（网页/Word/记事本）多次 Ctrl+C，每段内容自动入队，无需切换记事本
2查看列表置顶窗口实时显示所有复制项，序号即粘贴顺序（顶部=下一条）
3按序粘贴开启模式后，在目标文档连续 Ctrl+V，每次输出下一条并自动消除
4循环至空继续按 Ctrl+V 直至非置顶项全部输出完毕

操作与快捷键

开启/关闭「队列粘贴模式」点击顶部 队列粘贴：关 按钮，开启变 队列粘贴：开；模式开时 Ctrl+V 才会按序出队
按序出队粘贴模式开 + Ctrl+V（队列为空后自动恢复普通粘贴）
临时普通粘贴（逃生口）模式开时按住 Shift + Ctrl+V，本次不消耗队列
点选粘贴（普通项）单击列表项 → 粘贴到当前光标并从列表移除
点选粘贴（置顶项）双击置顶项 → 粘贴但保留在列表（可反复使用）
置顶 / 取消置顶点击项右侧 📌 置顶（移入置顶区）；置顶项点 📍 取消
删除单项点击项右侧 ✕
清空普通队列顶部「清空」按钮（置顶项不受影响，含所有分组）
新建分组顶部「+分组」按钮 → 输入名称。新建后自动设为活动分组，新复制内容归入此组
切换活动分组单击分组头（高亮 ● 标记）。活动组决定新复制内容归属；未选任何组则归入"未分组"
折叠/展开分组点击分组头左侧箭头（▶/▼）
重命名/删除/清空分组右键分组头 → 对应菜单项。删除分组时组内项移至未分组，不连带删除
移动项到其他分组右键列表项 →「移动到分组…」→ 选择目标分组（含"未分组"）。当前所在分组会勾选并禁用。右键不会触发粘贴
显示/隐藏窗口双击托盘图标，或托盘右键菜单
切换主题（暗/亮）顶栏右侧太阳/月亮图标按钮。默认暗色，点击切亮色，再点切回。不记忆，重启默认暗色
退出程序托盘右键 →「退出」（点窗口 ✕ 只是隐藏到托盘）
置顶项 Pin 的作用：在 Ctrl+V 自动出队流水线中被跳过、不消除， 只在你手动双击时才输出且保留——适合存放要反复粘贴的重要片段（如邮箱、签名、代码片段）。

读写分离 · 互不干扰

边粘边复制

粘贴未完成时仍可随时 Ctrl+C 追加新内容到队列末尾，复制与粘贴完全独立、并行不悖。

窗口常驻置顶

主窗口始终置顶且不抢焦点：点击列表项时目标应用（记事本等）保持焦点，粘贴直达光标处。

内容分组 · 跨文档归类收集

默认状态（未建任何分组）下，界面与行为与无分组时完全一致——所有内容在一个扁平列表中。分组是可选启用的，不破坏现有体验。
1建分组点顶部「+分组」命名。新建即自动成为活动分组（●标记）
2归类收集复制文档A内容→归入分组A；切到分组B后再复制→自动归入分组B
3同屏浏览所有分组同时可见，各带可折叠头(▶/▼)与项数。未分组项置顶显示
4按序输出开模式后 Ctrl+V 全局 FIFO 跨所有分组按复制时间先后输出，直至清空
分组作用范围仅普通队列。置顶区独立于分组，不归属任何分组
出队顺序全局 FIFO——跨所有分组+未分组按复制时间取最早，不受当前活动分组影响。活动分组只决定"新复制归到哪"
删除分组组内项自动移至"未分组"，不会被删除；删除活动分组后活动组重置为未分组
持久化不持久化。分组及内容仅本次运行有效，退出即清空（与队列行为一致）

运行与启动

直接运行src\MultiCopy\bin\Debug\net8.0-windows\MultiCopy.exe
从源码运行dotnet run --project src\MultiCopy
开机自启托盘右键 →「开机自启」勾选（写注册表 HKCU Run 项，无需管理员）
单实例已运行时再次启动会自动退出，不会多开

已知限制

  • 右键粘贴不走键盘钩子，不会触发出队——请用 Ctrl+V。
  • UAC 不对称：若目标应用以管理员权限运行而 MultiCopy 不是，钩子收不到其按键。需对管理员应用操作时，请以管理员身份运行 MultiCopy（修改 app.manifest 为 requireAdministrator）。
  • 控制台/部分游戏等非标准粘贴应用可能不响应模拟 Ctrl+V。
  • 仅采集纯文本（富文本/HTML/图片自动转为纯文本）。

Models/ClipboardItem、ClipboardGroup（分组）、ClipboardQueue（未分组+分组+置顶，全局FIFO出队）
State/AppState.cs全局标志位：模式开关、活动分组、防回环、防重入
Services/剪贴板服务、监听器、键盘钩子、粘贴执行器、托盘
ViewModels/ · MainWindow.xamlMVVM 视图模型与置顶列表 UI
