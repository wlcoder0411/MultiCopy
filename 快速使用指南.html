📋 MultiCopy 快速使用指南

多段连续复制 · 队列按序/选择性粘贴 · 文本+图片采集 · 置顶持久化 · 内容分组 · 全局热键 · 搜索过滤 · 暗亮双主题

核心流程

1连续复制在任意应用（网页/Word/记事本/截图工具）多次 Ctrl+C，每段文本或图片自动入队，无需切换记事本
2查看列表按 Alt+Z 调出窗口，实时显示所有复制项，序号即粘贴顺序（顶部=下一条）
3按序粘贴开启模式后，在目标文档连续 Ctrl+V，每次输出下一条并自动消除
4循环至空继续按 Ctrl+V 直至非置顶项全部输出完毕

操作与快捷键

调出/隐藏主窗口Alt+Z（全局热键，可在设置中自定义）v3.x
开启/关闭「按序粘贴模式」点击顶部 按序粘贴：关 按钮，开启变 按序粘贴：开；模式开时 Ctrl+V 才会按序出队
按序出队粘贴模式开 + Ctrl+V（队列为空后自动恢复普通粘贴）
临时普通粘贴（逃生口）模式开时按 Ctrl+Shift+V，本次不消耗队列
点选粘贴（普通项）单击列表项 → 粘贴到当前光标并从列表移除
点选粘贴（置顶项）双击置顶项 → 粘贴但保留在列表（可反复使用）
置顶 / 取消置顶点击项右侧 📌 置顶（移入置顶区）；置顶项点 📍 取消
删除单项点击项右侧 ✕
清空普通队列顶部「清空」按钮（置顶项不受影响，含所有分组）
搜索过滤顶部搜索框输入关键词，实时过滤列表。文本按内容匹配，图片按元数据匹配；Esc 清空搜索框v3.x
新建分组顶部「+分组」按钮 → 输入名称。新建后自动设为活动分组，新复制内容归入此组
切换活动分组单击分组头（高亮 ● 标记）。活动组决定新复制内容归属；未选任何组则归入"未分组"
折叠/展开分组点击分组头左侧箭头（▶/▼）
重命名/删除/清空分组右键分组头 → 对应菜单项。删除分组时组内项移至未分组，不连带删除
移动项到其他分组右键列表项 →「移动到分组…」→ 选择目标分组。右键不会触发粘贴
监控开关顶部「监控」开关按钮。关闭后完全静默待机（不采集 + 不拦截 Ctrl+V），与窗口可见性联动v3.x
切换主题（暗/亮）顶栏右侧太阳/月亮图标按钮。默认暗色，点击切亮色，再点切回。不记忆，重启默认暗色
设置对话框顶栏齿轮图标 → 自定义全局热键（修饰键+按键组合）v3.x
显示/隐藏窗口Alt+Z 全局热键，或双击托盘图标，或托盘右键菜单
开机自启托盘右键 →「开机自启」勾选（写注册表 HKCU Run 项，无需管理员；启动时仅驻留托盘不弹窗）v3.x
退出程序托盘右键 →「退出」（点窗口 ✕ 只是隐藏到托盘）
置顶项 Pin 的作用：在 Ctrl+V 自动出队流水线中被跳过、不消除， 只在你手动双击时才输出且保留——适合存放要反复粘贴的重要片段（如邮箱、签名、代码片段）。 置顶内容持久化保存到 %APPDATA%\MultiCopy\pinned.json，重启后不丢失。v3.x

文本与图片采集 v3.x

📄 文本采集

自动监听剪贴板，每次复制纯文本自动入队。含去重机制（500ms 时间窗口 + 内容比对），不会重复采集同一段文本。

🖼️ 图片采集

支持采集截图、复制的图片。图片格式按 CF_DIBV5 > PNG > CF_DIB 优先级读取。含去重（1500ms 时间窗口 + 尺寸+哈希双校验）。

队列容量最多保留 30 项（含文本与图片），超出自动移除最旧的非置顶项
超大图片超过 4096×4096 像素的图片会被拒绝采集，避免内存占用过高
图片展示列表中显示缩略图与来源应用名称
图片粘贴点选或按序粘贴时，图片以 CF_DIB 格式写入剪贴板，目标应用直接粘贴为图片

读写分离 · 互不干扰

边粘边复制

粘贴未完成时仍可随时 Ctrl+C 追加新内容到队列末尾，复制与粘贴完全独立、并行不悖。

窗口常驻置顶

主窗口始终置顶且不抢焦点：点击列表项时目标应用（记事本等）保持焦点，粘贴直达光标处。

内容分组 · 范围 FIFO 出队 v1.7+

默认状态（未建任何分组）下，界面与行为与无分组时完全一致——所有内容在一个扁平列表中。分组是可选启用的，不破坏现有体验。
1建分组点顶部「+分组」命名。新建即自动成为活动分组（●标记）
2归类收集复制文档A内容→归入分组A；切到分组B后再复制→自动归入分组B
3同屏浏览所有分组同时可见，各带可折叠头(▶/▼)与项数。未分组项置顶显示
4按序输出开模式后 Ctrl+V 按范围 FIFO 输出（见下表规则）
分组作用范围仅普通队列。置顶区独立于分组，不归属任何分组
出队顺序（范围 FIFO）活动分组非空→仅该分组内按 CreatedAt 最早出队；活动分组为空（null）→全局 FIFO，跨所有分组+未分组按复制时间最早出队
删除分组组内项自动移至"未分组"，不会被删除；删除活动分组后活动组重置为未分组
持久化分组及普通内容不持久化，退出即清空。仅置顶项持久化跨会话保留

运行与启动

直接运行双击 MultiCopy.exe（单文件自包含，无需安装 .NET 运行时）
从源码运行dotnet run --project src\MultiCopy
开机自启托盘右键 →「开机自启」勾选（写注册表 HKCU Run 项，--minimized 参数仅驻留托盘不弹窗）
监控开关关闭监控后完全静默待机：不采集剪贴板内容、不拦截 Ctrl+V。窗口隐藏时自动关闭，显示时自动开启
单实例已运行时再次启动会自动退出，不会多开

数据存储

置顶项%APPDATA%\MultiCopy\pinned.json（含文本与图片，原子写入）+ %APPDATA%\MultiCopy\images\<id>.png（图片文件）
设置%APPDATA%\MultiCopy\settings.json（自定义热键等配置）
普通队列仅内存，不持久化，退出即清空

已知限制

  • 右键粘贴不走键盘钩子，不会触发出队——请用 Ctrl+V。
  • UAC 不对称：若目标应用以管理员权限运行而 MultiCopy 不是，钩子收不到其按键。需对管理员应用操作时，请以管理员身份运行 MultiCopy（修改 app.manifest 为 requireAdministrator）。
  • 控制台/部分游戏等非标准粘贴应用可能不响应模拟 Ctrl+V。
  • 图片搜索仅按元数据匹配（来源应用、时间），不含 OCR 文字识别。
  • 主题选择不持久化，每次启动默认暗色主题。

Models/ClipboardItem（文本/图片）、ClipboardGroup（分组）、ClipboardQueue（队列+范围FIFO出队）
State/AppState.cs全局标志位：模式开关、活动分组、监控开关、防回环、防重入
Services/剪贴板监听/服务、键盘钩子、粘贴执行器、热键服务、托盘、主题、置顶持久化、设置存储
ViewModels/MainViewModel（MVVM 视图模型）
Views/主窗口 + 设置对话框 + 分组输入对话框 + 支持作者对话框
Themes/Dark/Light 主题、图标资源、通用样式（13 个语义 Token）
Infrastructure/转换器、模板选择器、图标工厂、热键格式化器
